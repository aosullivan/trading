## Trust Boundaries

- Frozen market fixtures under `tests/fixtures/focus_basket/` are trusted test inputs and must not silently drift from the documented spec.
- `/api/chart` is the authoritative scoring surface; benchmark code must not bypass it with a separate calculation path.
- The promoted baseline artifact will be used to block regressions later, so accidental edits to the spec or floors are integrity-sensitive.

## STRIDE Register

| threat_id | category | component | disposition | mitigation_plan |
|-----------|----------|-----------|-------------|-----------------|
| TM-03-01-01 | tampering | benchmark fixture spec and pinned floors | mitigate | Keep the source-of-truth in one JSON spec, document regeneration steps, and fail tests when fixture names or pinned metrics drift. |
| TM-03-01-02 | repudiation | ratchet promotion evidence | mitigate | Write a human-readable baseline summary that records the exact tickers, request params, score formula, and pinned metrics used for the promoted baseline. |
| TM-03-01-03 | denial_of_service | benchmark tests depending on live Yahoo data | mitigate | Regenerate CSVs with a script, but make the pytest benchmark read only committed CSV fixtures and patched downloads. |
</threat_model>
